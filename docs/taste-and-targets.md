## Hard exclusions (never surface)

- **Russian** (Poljot, etc.) — no.
- **Chinese** watches / Chinese-based movements / Chinese clones — no.
- **Japanese movements** (Miyota, Citizen, Seiko incl. **Seiko chronographs**) — no; doesn't enjoy working on them. **Only** possible exceptions: vintage **Credor, Grand Seiko, King Seiko**.
- Default to **Swiss-made**.
- **Pocket watches** — wristwatches only. *(Auto-filter: constrain searches to the eBay **Wristwatches** category `31387` and/or drop titles containing "pocket watch". Bonus: the category constraint also strips out straps/boxes/loose-movement accessory listings.)*
- **Quartz** — **mechanical only** (automatic + manual). Removes a huge slice of noise.
- **Smartwatches / digital / fitness** — Apple, Garmin, Fitbit, digital G-Shock, etc.
- **Parts-only listings** — "movement only," "dial only," "case only," "parts lot." *(Distinct from a for-repair **watch**, which IS wanted.)*
- **Fashion / mall brands** — Fossil, Michael Kors, Invicta, Stuhrling, Diesel, Guess, Armani, Daniel Wellington, Swatch, **Bulova**.
- **Replica tells** — drop titles with "replica," "AAA," "homage." ⚠️ **Keep "tribute"** (legit, e.g. IWC Tribute to 3705).
- **Redial tells** — "redial," "refinished dial," "aftermarket dial" (original dial = most of the value).
- ❌ **No flat minimum-price floor** — deliberately. Real auction gems open at $1 (e.g. the Wakmann Venus 188). "Too cheap = fake" stays *relative to value*, never an absolute cutoff.

## Movement philosophy — grade matters as much as the caliber

He's a "movement person" — values high-grade, high-complication, niche movements.
**The same base caliber varies wildly in desirability by finish/grade:**
- Base **Valjoux 7750** = "meh" / beater (still useful, always works).
- **Chronometer-grade 7750** (e.g. Breitling) = higher, more accurate → liked.
- **In-house-finished 7750** (vintage/neo-vintage **IWC**: own escape wheel, optimized pallet-fork angle, own mainspring/gears, gold vs. chrome-plated) = prized.
- → Scoring must weight **brand+grade**, not just the base caliber. Vintage/neo-vintage **IWC (~1980s–early 2000s, pre-Richemont)** is a personal high-value zone.

Also loves: **complications** (triple-calendar chronographs, etc. — harder = more
interesting to service), and **vintage automatics with date** (his JLC K881/883).

## Feel / sizing (reaffirmed)

- Chronographs **>36mm**, **solid stainless steel**, substantial — too-tiny watches feel like a toy (the Landeron-148 Bovet was too small despite a liked movement); the **41mm 7750 Breitling** is the sweet spot.
- **Screw-in crown valued but not required** (Navitimer has none and he loves it).
- **Age floor ~1960** — nothing older (too fragile/old-tech to service). Wants **shock protection** (Incabloc), **movable stud carrier**, robust hairspring. See playbook §2d.

## Search recipes (his actual eBay technique)

The wide-net trawls to replicate, all starting from `q = "chronograph watch"`:
- **A — contested auctions:** Filter **Auction** + sort **number of bids (most first)**. The most-bid chrono auctions; knowledgeable buyers have already noticed → something interesting usually surfaces.
- **B — as-is auctions:** add Condition = **"For parts or not working"** (eBay's newer filter) + Auction + sort most bids.
- **C — as-is, all listings:** like B but drop the Auction filter (BIN + auction).

**eBay API constraint to flag:** the Browse API supports `buyingOptions:{AUCTION}` and
`conditions` (for-parts = condition 7000), but **does NOT sort by bid count** (only the
deprecated Finding API did). So we fetch the as-is chrono auction set and **rank
client-side** — by bidCount if returned, else by our own reference-DB value score.
"Most bids" is really a proxy for "knowledgeable buyers noticed it," which our scoring
partially replaces.

## Seller gate (confirmed)

Full two-dimensional model in [watch-playbook.md §1B](watch-playbook.md). Short version:
prefer 100%; **< 20 sales = skip**; high-volume sellers OK at 95%; **< 90% always cut**;
and seller *type* (watch-focused vs fashion/junk-volume vs estate/incidental) matters
more than raw numbers. Estate/incidental sellers = low trust but highest gem potential.

## What eBay exposes per listing (for the scorer)

Confirmed via live item-detail calls. **Two tiers:**

- **Search (`item_summary/search`)** — cheap, bulk: title, price (BIN) / currentBidPrice (auction), condition, seller (feedback %/score, username), itemWebUrl, image, buyingOptions, itemLocation. **No size/movement/aspects.**
- **Item detail (`getItem`, separate 5k/day quota)** — rich **item specifics** *when the seller filled them in*: **Case Size** ("40 mm"), **Movement** ("Automatic" / "Mechanical (Manual Winding, **Calibre 1861**)" — sometimes names the caliber), **Case Material**, **Caseback** (Solid/…), **With Box/Papers**, **Reference Number**, **Model**, **Brand**, **Year Manufactured**, **Country of Origin** (Swiss check), **Features** (Chronograph…); plus top-level **authenticityGuarantee**, **condition**, **returnTerms**, **description**, **additionalImages**.

→ Many gates become **structured & reliable** when present: quartz-exclude (Movement), size >36mm (Case Size), box/papers, solid-steel vs plated (Case Material), screw/solid caseback, Swiss (Country), brand-exclude (Brand), age floor (Year), + the Authenticity Guarantee trust signal. Caliber sometimes free.

→ **Caveat (the gem case):** aspects are only as good as the seller's data entry. Dealer/modern listings are richly filled; the **underpriced as-is/estate gems are often sparse/blank** → fall back to **title parsing + reference→caliber DB + photos + the user's eye.**

→ **Flow:** search wide & cheap → `getItem` only on scored candidates → pull these fields → final score + photos to phone.
